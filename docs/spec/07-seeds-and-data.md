# 07 Seeds And Data

## Chapter 7.1 Seed Directory Contract

Seeds live under:

```text
seeds/{seedName}
```

Seed names must match:

```text
^[A-Za-z0-9_-]+$
```

An environment can only be created when:

- `seeds/{seedName}` exists.
- `seeds/{seedName}/mongodb` exists.

The `mongodb` folder is generated by `scripts/prepare-seeds.sh`.

## Chapter 7.2 Default Seed Files

The default seed is:

```text
seeds/default
```

Top-level JSON files:

- `users.json`
- `settings.json`
- `tenants.json`

The prepare script imports each top-level JSON file into MongoDB collection names based on file basename:

- `users.json` -> `users`
- `settings.json` -> `settings`
- `tenants.json` -> `tenants`

## Chapter 7.3 Default Seed Users

`seeds/default/users.json` contains environment application users:

- `admin@example.local`, role `admin`
- `arsenii@example.local`, role `admin`
- `editor@example.local`, role `editor`
- `viewer@example.local`, role `viewer`

These records are environment seed data. They do not provision central Composer login users.

## Chapter 7.6 Seed Preparation Flow

`scripts/prepare-seeds.sh`:

1. Checks `jq` and Docker availability.
2. Iterates every directory under `seeds`.
3. Creates `{seed}/media` if missing.
4. Clears `{seed}/mongodb` using a temporary Mongo image.
5. Starts a temporary MongoDB container with `{seed}/mongodb` mounted as `/data/db`.
6. Waits for MongoDB to respond to ping.
7. Touches and drops `__primarie_seed_ready`.
8. Imports every top-level `*.json` file using `mongoimport --drop`.
9. Stops and removes the temporary container.
10. Runs chmod normalization on `{seed}/mongodb`.

Default database:

```text
primarie
```

Default Mongo image:

```text
mongo:7.0.12
```

## Chapter 7.7 Seed Copy Into Environments

During `environment.prepare`, `scripts/prepare-env.sh` copies:

- `{hostSeedsDir}/{seedName}/mongodb` into `{runtimePath}/data/mongodb`
- `{hostSeedsDir}/{seedName}/media` into `{runtimePath}/data/media` when present

It removes previous target Mongo/media data before copying.

## Chapter 7.8 Seed Services In API Source

There are TypeScript seed helper classes:

- `SeedService.copySeed`
- `MongoSeedDumpService.createDump`

Current active environment preparation uses Bash scripts, not these classes.

`MongoSeedDumpService` builds a `seed-data.js` and copies JSON files into a destination path. This appears to support the older `mongo-dump` template path.

## Chapter 7.9 Central Auth Data

The central API authenticates against MongoDB collection `users` in database `primarie`.

Required central auth user shape:

```ts
{
  email: string;
  name: string;
  password: string;
}
```

`password` must be a bcrypt hash.

No source script currently provisions central auth users from `seeds/default/users.json`.
